This task involves finishing the implementation of copy-propagation and constant-folding found in CopyConstPropFold.sml.

Constant Folding and Copy/Constant Propagation Background

Constant Folding is an optimization that evaluates expressions that yield constant values whenever possible at compile time rather than runtime. For example, expresssions involving only constants can always be folded: such is the case with expression 2+3 that can be directly evaluated by the compiler and replaced by its result, 5. Furthermore, constant folding can also operate in cases in which the folded expression does not contain only constants by exploiting algebraic identities. For example, expression e+0 can always be replace with the expression e, because 0 is the neutral element for addition.

In copy/constant propagation, the compiler reacts to bindings such as let x = a or let y = 2, which merely "copy" a variable or constant, and replace subsequent uses of x and y with a and 2 respectively. In constant folding, we simplify expressions whose value can be determined at compile-time. In the running example, once we have performed the substitution y |-> 2 in the expression y-2, we obtain 2-2, which, since it contains only constants, can be evaluated at compile-time to 0 (by constant folding). In the original expression (x + 2) * (y - 2) we thus produce (x + 2) * 0, which can again be constant-folded to 0.

fun int f(int x, int y) =             fun int f(int x, int y) =
  (x+2) * (y-2)                         (x+2) * (y-2)

fun int main() =                      fun int main() =
  let a = read(int) in                  let a = read(int) in
  let b =                 ---->         let b =
    let x = a in                          let x = a in
    let y = 2 in                          let y = 2 in
    (x + 2) * (y - 2)                     0
  in                                    in
  b                                     b

Furthermore, if carefully implemented, copy propagation can also normalize nested let bindings, such that it exposes more propagation oportunities:

fun int f(int x, int y) =             fun int f(int x, int y) =
  (x+2) * (y-2)                         (x+2) * (y-2)

fun int main() =                      fun int main() =
  let a = read(int) in                  let a = read(int) in
  let x = a in            ---->         let x = a in
  let y = 2 in                          let y = 2 in
  let b = 0 in                          0
  b

Note that the program still contains the now-unused bindings for a, x and y - these will be removed by further dead-code elimination, yielding in the end the program

fun int main() =
  let a = read(int) in
  0

The binding of a cannot be removed because read(int) has side effects, but this is only of concern to the dead code removal pass and not part of copy propagation or constant folding.

Fundamental Implementation

The core of the implementation is the following function:

copyConstPropFoldExp : SymTab.SymTab Propagatee -> Exp -> Exp

This function takes as input a symbol table and an expression, and produces an optimised (possibly unchanged) expression. The symbol table maps variable names to Propagatee values, which are defined as follows:

datatype Propagatee = ConstProp of Value
                    | VarProp of string

Not all variables in the program are present in the symbol table - only those who are let-bound to either a constant (ConstProp) or a variable (VarProp). Here, then, is the basic mechanism of copy propagation: Whenever we encounter a usage of a variable with an entry in the symbol table, we replace the variable with its definition.

Copy/Constant Propagation

The copyConstPropFoldExp function uses case analysis on the expressions constructor, and copy/constant-propagation is fully implemented in the cases for Var, Index, and Let expressions. For the purpose of copy/constant-propagation, the other nodes just apply copyConstPropFoldExp recursively on the node’s subexpressions so that all program (sub)expressions are optimised.

fun copyConstPropFoldExp vtable e =
    case e of
      (* Copy/Constant propagation is handled entirely in the following
          three cases for variables, array indexing, and let-bindings. *)
        Var (name, pos) =>
           ... fill-in the blanks (1) ...

      | Index (name, e, t, pos) =>
           ... fill-in the blanks (2) ...

      | Let (Dec (name, e, decpos), body, pos) =>
        let val e' = copyConstPropFoldExp vtable e
        in case e' of
               Var (varname, _) =>
                   ... fill-in the blanks (3) ...

             | Constant (value, _) =>
                   ... fill-in the blanks (4) ...

             | Let (Dec bindee, inner_body, inner_pos) =>
                   ... fill-in the blanks (5)

             | _ =>
               let val body' = copyConstPropFoldExp vtable body
               in Let (Dec (name, e', decpos), body', pos)
               end
        end

Your implementation consists of filling in the blanks (1), (2), (3), (4), and (5).

  • (1) current expression is a variable, for example Var("x", p). We need to check whether this variable is a candidate for replacement, i.e., wether we are in a case such as let x = y in ... x ... and if so we need to replace the use of x with y. This checking requires to perform a lookup in vtable:

    • If the lookup succeeds with SOME (VarProp "y"), i.e., copy propagation case, then the expression result becomes the new variable y, i.e., the result is Var("y", p).

    • If the lookup succeeds with SOME (ConstProp v), i.e., constant propagation case, then the expression result should be constant expression v.

    • Otherwise the expression result is the original expression, i.e., Var("x", p).

  • (2) current expression is an index, for example Index("x",e,t,p). Similarly as above we need to check whether we are in a replacement case, such as for example let x = y in ... x[i] ..., and if so to return y[i] instead of x[i]:

    • apply copy/constant propagation (and folding) to the index expression e,

    • treat variable x similarly as above, i.e., find its copy/constant replacement if any

    • and form the result index expression from the results of the previous two steps.

  • The case when the current expression is a let binding, for example let x = e in body is the one that fills in/updates the vtable. We first apply copy/constant propagation and folding to e, and then we pattern-match the resulted expression e'

    • (3) if e' is a variable named "y" then we are in the copy propagation case let x = y in body, and hence we need to add a new x |-> VarProp "y" binding to the vtable, and to recursively optimize body under the extended vtable.

    • (4) if e' is a value v, then we are in a constant propagation case,

      1. similar to (3) but add a ConstProp binding.

    • (5) correspond to tricky cases such let x = (let y = 4 in 5+y) in body. In this case e is let y = 4 in 5+y, and e' (the optimized e) is let y = 4 in 9. Note that while x is let-binded to constant value 9 this is obscured by the nested binding of y, i.e., we have let x = (let y = 4 in 9) in body. What we can do in this case is to:

      • First, normalize the nested let binding, i.e., transform let x = (let y = 4 in 9) in body into the semantically equivalent expression let y = 4 in let x = 9 in body, and

      • Second, recursively optimize (by copyConstPropFoldExp) the expression obtained in the previous step.

Constant Folding

For the implementation of constant folding, you will have to implement the language constructs that you have introduced in Task 1: multiplication, division, logical and/or, logical and integer negation. Constant folding performs algebraic simplification of scalar expressions. For demonstration we look at the provided code for addition:

fun copyConstPropFoldExp vtable e =
    case e of ...
      | Plus (e1, e2, pos) =>
        let val e1' = copyConstPropFoldExp vtable e1
            val e2' = copyConstPropFoldExp vtable e2
        in case (e1', e2') of
               (Constant (IntVal x, _), Constant (IntVal y, _)) => Constant (IntVal (x+y), pos)
             | (Constant (IntVal 0, _), _)                      => e2'
             | (_, Constant (IntVal 0, _))                      => e1'
             | _                                                => Plus (e1', e2', pos)
        end

The code above calls recursively copyConstPropFoldExp on subexpressions e1 and e2 to perform propagation and constant folding recursively, resulting in optimised subexpressions e1' and e2'. Then, we see whether we can simplify the Plus expression itself:

  • if both e1' and e2' are constant values than we just create a new value (expression) which is the sum of the two,

  • otherwise if e1' (e2') is constant 0 then the result is e2' (e1'), because 0+e2' == e2'. If e2' happens to be a variable than further copy propagation might be enabled.

  • otherwise a new (optimised) expression is created Plus (e1', e2', pos).

In a similar way your task is to implement *, /, &&, ||, ==, not. For example, for a multiplication expression: e1' * e2', where e1' and e2' are the recursively optimised subexpression, further simplifications can be done in several cases:

  • if e1' and e2' are constant values then the result will be the multiplication value,

  • if e1' (e2') is value 1 then the result is e2' (e1') because 1 * e2' = e2',

  • if if e1' (e2') is value 0, then you cannot simplify to the constant value 0, because the other operand may have side effects.

  • otherwise the optimised result is Times (e1', e2', pos).

  • A similar rationale can be applied to /, &&, ||, =, not.
